Recommended Usage of SPARK Ada

  • For terminology such as directory, ~/bin etc see the short file Basic Terminology in Linux
  • See basic usage of SPARK Ada for basic usage instructions.
  • It is recommended that for small projects you
    • have all files (file.ads and file.adb) belonging to it in one directory,
    • add the file main.gpr to it
  • SPARK Ada will generate some files automatically, especially there will be a subdirectory gnatprove.
  • When you open with gps the main.gpr file provided, all ads and adb files in the directory where the main.gpr file is located will belong to the current SPARK Ada and nothing else
    • Especially no files in any subdirectories will be added.
  • It is recommended that when opening gps to open a terminal, navigate using the command

    cd dirname

    for changing the directory to dirname
    and

    cd ..

    for moving up one level in the directory hierarchy
    until you have reached the directory with a project containing your file main.gpr.
    Then start gps on main.gpr using

    gps main.gpr &

  • Please open any files associated with your project using the side menu, and not using the file manu from the top.
    There is a file icon in the side menu with a "." (which refers to the home directory of the project) and if you click on it you can access the files in the current directory.
  • If you open files directly from the file menu, and they are not in the same directory as your gps file, then these files will not be checked using SPARK Ada, since SPARK Ada will only check files associated with your current gpr file.
  • Note that file names need to be in lower case and should not start with a digit otherwise gps will ignore those file.
  • Before deleting/renaming/editing files without using gps it is best to close down gps first and then reopen it again. Otherwise you get synchronisation problems with gps.
  • For creating a new project the best is to create a new directory, and copy from an existing example (for instance the ada examples from the lecture) the ads and adb fiels to your directory, possibly rename them, and then edit them to suit your needs.
