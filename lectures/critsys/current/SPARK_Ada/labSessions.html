Instructions for Linux Machines

  • Download the git responsiory as described in instructions for downloading the gt Repository for CSC313/CSCM13
  • Now we want to explore some examples.
    • It is important to leave the git repository untouched. Then you can update it by running
      git pull
      rather than having to download the whole repository again and again.
  • We first work on the example code for testing the installation.
    • It is located at
      ~/git/criticalhighintegritysystems/exampleCodeForTesting/
      This code is mainly used to check whether everything is set up correctly
    • Copy that folder to a location in your file system where you want to work on it (outside the git repository)
    • Now you can follow the Instructions so that you can use SPARK Ada (and why3) from the File Manager
    • Once you have done so, study basic usage instructions of SPARK Ada
    • Now you can look at Examples for getting started
    • Check out SPARK Ada code examples (including Examples from the lectures and other sources)
    • The lab sheet and the coursework can be found under Canvas.
    • Additional information
      • Cheat Sheet by Olga Petrovska
      • Some basic instructions about using Linux and Emacs can be found here
      • Instructions for setting up machines in the Linux Lab for using SPARK Ada and why3 from command line
